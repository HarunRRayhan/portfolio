# Production AI Code Review — Publish Order Checklist

## Goal
Publish the canonical article on `harun.dev` first, then syndicate adapted copies in a controlled order so the site keeps SEO authority and the external platforms still get indexed.

## Publish order
1. Publish the canonical post on `harun.dev`
2. Verify the live URL renders correctly and is not draft-only
3. Submit or post the adapted Dev.to version
4. Publish the Medium version
5. Publish the HackerNoon version
6. Post the LinkedIn teaser with the canonical link
7. Share a short discovery note on daily.dev if desired
8. After indexing has settled, optionally publish one more republish if we still want one

## Pre-publish checks
- [ ] Canonical article is live on `harun.dev`
- [ ] The final title is correct
- [ ] The article is not still marked `draft: true`
- [ ] The canonical URL points to the site post
- [ ] The article still reads as a review-layer companion, not a duplicate of earlier AI/AWS posts
- [ ] The site build passes

## Platform adaptation rules
- [ ] Keep the opening paragraph different per platform
- [ ] Keep the closing paragraph different per platform
- [ ] Keep the canonical link back to `harun.dev`
- [ ] Avoid byte-for-byte duplication where the platform does not support canonical URLs
- [ ] Keep the human-in-the-loop framing strong

## Platform notes
- **Dev.to**: best for broad technical discovery and comments
- **Medium**: good for reach if the copy is tightened
- **HackerNoon**: good for technical audience visibility
- **LinkedIn**: use as a teaser, not a long duplicate
- **daily.dev**: use as distribution/awareness, not the canonical home

## Final verification
- [ ] Canonical post live
- [ ] External copies published or scheduled
- [ ] Each syndication copy points back to the canonical article
- [ ] No duplicate-content ambiguity remains in the copy
- [ ] Social teaser posted
